## Spacing Scale

Declared values (multiples of 4 only). Derived from direct measurement of existing StyleSheet values across `TripCompletionScreen.tsx`, `FindingDriverModal.tsx`, and `ActiveTripSheet.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon touch padding, close button tap area padding |
| sm | 8px | Inline element gaps, label-to-value gap within summary rows |
| md | 16px | Default element padding, card internal padding, button vertical padding |
| lg | 24px | Section padding (header paddingHorizontal), footer padding, modal content margins |
| xl | 32px | Screen horizontal padding (paddingHorizontal: 32), between card and CTA button |
| 2xl | 48px | Not currently used in existing components; reserved for major section breaks |
| 3xl | 64px | Not currently used; reserved for splash-level full-screen centering offsets |

**Exceptions:**
- Touch targets (buttons): minimum `paddingVertical: 16` to meet 44pt touch target on iOS
- Modal drag handle area: system-rendered by iOS `pageSheet` presentation style — no custom spacing needed
- LoadingOverlay (new component): full-screen modal, no spacing tokens apply to the overlay itself
- Toast notification (new component): `react-native-toast-message` default positioning — do not override; library manages its own safe area insets

**Source:** Measured from existing codebase StyleSheet values (HIGH confidence)

## Component Inventory

New components introduced in Phase 6. Each must match the existing StyleSheet patterns.

### LoadingOverlay (new — D-05)

**Purpose:** Replace all 15 per-screen `ActivityIndicator` instances with one shared overlay.

**Visual contract:**
- Full-screen modal, `transparent: true`, `animationType: "none"`
- Semi-transparent black scrim: `backgroundColor: 'rgba(0,0,0,0.4)'`
- Centered `ActivityIndicator` size "large", color `#1565C0` (accent)
- Optional `message` text below spinner: 14px, weight 400, color `#FFFFFF`
- No border radius, no card — raw overlay only
- `visible` prop controls display; no animation on show/hide (snappy, not distracting)

**Props interface:**
```typescript
interface LoadingOverlayProps {
  visible: boolean;
  message?: string;
}
```

**Migration targets (15 screens confirmed in RESEARCH.md):**
- `app/_layout.tsx` — replaced entirely by splash screen hold (D-08); remove ActivityIndicator
- `app/(auth)/index.tsx`, `app/(admin)/index.tsx`, `app/(admin)/drivers.tsx`, `app/(admin)/company-setup.tsx` — replace per-screen ActivityIndicator with `<LoadingOverlay visible={loading} />`

---

### ToastProvider (new — D-06)

**Purpose:** Replace all `Alert.alert` error calls with auto-dismiss inline toasts.

**Visual contract:**
- Use `react-native-toast-message` defaults — do not customize the toast UI for this phase
- `type: 'error'` for Firebase errors (red left bar, library default)
- `type: 'success'` for confirmation messages if needed
- `visibilityTime: 3000` (3 seconds) for all toasts
- Placement: default (top of screen, below status bar)
- `<Toast />` component placed as LAST child of `GestureHandlerRootView` in `app/_layout.tsx`

**Call site pattern:**
```typescript
Toast.show({ type: 'error', text1: 'Could not load drivers', visibilityTime: 3000 });
```

**Migration targets (confirmed in RESEARCH.md):**
- `app/(commuter)/index.tsx`, `app/(driver)/index.tsx`, `app/(admin)/drivers.tsx`
- `components/FindingDriverModal.tsx`, `components/ActiveTripSheet.tsx`,
  `components/RequestServiceSheet.tsx`, `components/CommuterTripSheet.tsx`

### ErrorBoundary (new — D-07)

**Fallback UI visual contract:**
- Full-screen centered layout: `flex: 1, alignItems: 'center', justifyContent: 'center', padding: 32`
- Background: `#FFFFFF` (dominant)
- Icon: `Ionicons name="warning-outline"` size 48, color `#FF3B30` (destructive)
- Heading: "Something went wrong" — 22px, weight 700, color `#000000`

## Copywriting Contract

All copy derived from Phase 6 requirements (SEC-02, SEC-03, SEC-04) and existing in-app copy patterns.

| Element | Copy |
|---------|------|
| Primary CTA — LoadingOverlay message (optional) | "Loading..." (only shown when operation takes >500ms; default is spinner only) |
| Toast — Firebase read failure | "Could not load data. Check your connection and try again." |
| Toast — Firebase write failure (generic) | "Save failed. Please try again." |
| Toast — Auth error (sign in) | "Sign in failed. Check your email and password." |
| Toast — Cancel request failure | "Could not cancel your request. Please try again." |
| Toast — Request cancelled (status event) | "Request cancelled. Please try again." |
| Toast — Trip not found after accept | "Something went wrong finding your trip. Please try again." |
| Error boundary — generic fallback heading | "Something went wrong" |
| Error boundary — generic fallback body | "Please restart the app." |
| Error boundary — map fallback heading | "Map unavailable" |
| Error boundary — map fallback body | "The map could not be loaded. Core functionality is still available." |
| Destructive action — Cancel Request (commuter) | Button label: "Cancel Request" — no confirmation dialog; tapping immediately cancels (existing pattern, D-06 note: replace surrounding Alert.alert with toast on error only) |
| Destructive action — Cancel Job (driver) | Button label: "Cancel Job" — uses existing confirmation dialog already in ActiveTripSheet; do NOT migrate this specific confirmation from Alert.alert to toast (confirmation dialogs remain as Alert.alert; only error alerts migrate to toast) |

**Copywriting rules for this phase:**
1. Error toast `text1` must name the specific operation that failed, not a generic "Error occurred."
2. Error toast `text2` (sub-text) always provides a next step: "try again", "check connection", or "restart the app."
3. Loading overlay `message` is optional and omitted for fast operations (<500ms typical); include only for slow operations like Firestore batch writes.
4. Do not use "Oops" or exclamation marks in error states.

**Source:** Adapted from existing FindingDriverModal copy + SEC-03 requirement (HIGH confidence)

## Phase-Specific Interaction Notes

### Splash Screen Hold (SEC-04)

The existing `ActivityIndicator` in `app/_layout.tsx` is replaced by `expo-splash-screen`. The user never sees a loading spinner at startup — the native splash screen holds until `loading` from `useAuth()` resolves to `false`. There is no interim UI state to design for.

**Interaction contract:**
- `SplashScreen.preventAutoHideAsync()` called at module level in `_layout.tsx` (before component renders)
- `SplashScreen.hideAsync()` called in `useEffect` with `[loading]` dependency, only when `loading === false`
- `return null` while `loading === true` — splash screen remains visible; no React content rendered
- After `hideAsync()`: immediate redirect via `<Redirect>` to the correct role-based screen

### Error Boundary Placement

Two error boundaries wrap different parts of the tree:
1. **Root ErrorBoundary** — wraps `<RootLayoutNav>` in `_layout.tsx`; catches crashes in nav-level rendering
2. **MapErrorBoundary** — wraps `<MapView>` in `app/(commuter)/index.tsx` and `app/(driver)/index.tsx`; catches Google Maps SDK render failures

Error boundaries are CLASS components (React 19 requirement for `componentDidCatch`). They do not use hooks.

### Toast Position

`<Toast />` is the last element inside `<GestureHandlerRootView>` in `_layout.tsx`, after the `<Stack>` navigator. This ensures toast renders above all modals (FindingDriverModal, ActiveTripSheet) without z-index conflicts.
