## Component Inventory

### Modified: FindingDriverModal.tsx

**Current states handled:** `searching`, `accepted`, `cancelled`
**New state to handle:** `no_drivers`

#### no_drivers State Layout

Replaces the animated spinner content area while keeping the existing header and footer structure unchanged.

```
[ Header: "Finding Driver" title + close button ]
[ Divider ]
[ Content area — centered ]
  [ Icon: ⚠️ at 48px — existing emoji pattern ]
  [ Heading: "No Drivers Available" — 22px bold, color #000 ]
  [ Body: "All nearby tow yards are currently busy. Please try again in a few minutes." — 15px normal, color #666, textAlign center ]
[ Footer ]
  [ "Try Again" button — same border style as existing Cancel button ]
```

The "Try Again" button calls `onCancel()` which dismisses the modal. The commuter is then returned to the map with the request button re-enabled. "Try again" means re-submitting a new request — no automatic retry.

#### Updated driver-found confirmation

When `request.status === 'accepted'`, the existing `Alert.alert` or direct navigation fires. No UI text change is needed inside `FindingDriverModal` itself — the copy update ("from [Company Name] is on the way") lives on the `CommuterTripSheet` active trip screen, which is Phase 3 scope. This spec records the intent but defers that copy to Phase 3 planning.

**Phase 2 scope for FindingDriverModal:** Add `no_drivers` handler only.

## Copywriting Contract

Source: CONTEXT.md decisions + existing component copy patterns.

| Element | Copy |
|---------|------|
| Primary CTA (commuter) | "Request Roadside Assistance" (existing, unchanged) |
| Searching state heading | "Finding Driver" (existing, unchanged) |
| Searching state body | "Searching for available drivers..." (existing, unchanged — commuter sees this throughout all dispatch stages per CONTEXT.md decision) |
| no_drivers heading | "No Drivers Available" |
| no_drivers body | "All nearby tow yards are currently busy. Please try again in a few minutes." |
| no_drivers CTA | "Try Again" |
| Driver-found confirmation (Phase 3 scope) | "[Driver Name] from [Company Name] is on the way" |
| Error state (Firebase op failure) | "Something went wrong. Could not find your trip. Please try again." (existing Alert pattern, unchanged) |
| Destructive — cancel request | Alert title: "Cancel Request" / message: "Are you sure you want to cancel your request?" / buttons: "Keep Waiting" (cancel) + "Cancel Request" (destructive) |
| Destructive — driver decline | No commuter-facing copy; commuter continues to see "Searching..." per CONTEXT.md decision |

Destructive confirmation approach:
- Cancel request: iOS `Alert.alert` with two-button pattern — default button dismisses, destructive button calls `cancelRequest()`. Use `style: 'destructive'` on the cancel button per existing sign-out pattern.
- Driver decline: handled server-side; commuter sees no change in modal copy.

## Interaction Contract

### FindingDriverModal — no_drivers transition

| Event | Trigger | Animation | Duration |
|-------|---------|-----------|----------|
| no_drivers state entered | `request.status === 'no_drivers'` in useEffect | Fade content area: opacity 1→0 over 200ms, then render no_drivers content, opacity 0→1 over 200ms | 400ms total |
| "Try Again" tapped | User tap | Modal slides down (existing `animationType="slide"`) | Platform default |
| Animated dots loop | `visible === true` and `status !== 'no_drivers'` | Stop loop when no_drivers state entered | Immediate |

### Existing interactions (unchanged)

| Interaction | Behavior |
|-------------|----------|
| FindingDriverModal — searching | Three-dot pulse animation, staggered 200ms, loops until status changes |
| FindingDriverModal — accepted | Auto-navigates to CommuterTripSheet, modal dismissed |
| FindingDriverModal — cancelled | Alert shown, modal dismissed via onCancel() |
| RequestPopup — accept | Blue primary button, disabled+opacity while isLoading |
| RequestPopup — decline | Outlined button, no destructive color (declining is normal driver behavior) |
| RequestPopup — timeout | Auto-calls onDecline() at 0 seconds; progress bar turns red at <=5s |
| Driver online toggle | Switch component, blue track when online, grey when offline |
