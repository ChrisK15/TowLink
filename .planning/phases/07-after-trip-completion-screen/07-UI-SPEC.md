## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label/value inline separation |
| sm | 8px | Compact row padding, between summary rows |
| md | 16px | Default horizontal/vertical element padding |
| lg | 24px | Section padding, container horizontal insets, footer padding |
| xl | 32px | Content area horizontal insets (centered content), margin below checkmark |
| 2xl | 48px | Vertical spacing between checkmark icon and header text |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Checkmark icon: 80px size (functional icon size, not a spacing token — matches Ionicons size prop)
- Done button paddingVertical: 16px (touch target height = 16 + 16 + line height ≈ 52px, meets 44px minimum)
- Summary card rows: 12px vertical padding per row (between sm 8 and md 16 — matches existing infoRow pattern in ActiveTripSheet)

**Source:** Direct measurement from FindingDriverModal.tsx (paddingHorizontal: 24, paddingTop: 24, footer padding: 24) and ActiveTripSheet.tsx styles. 8-point scale confirmed as existing standard.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Done" — single word, bottom of screen (D-05) |
| Driver role header | "Job Complete!" (D-08) |
| Commuter role header | "Trip Complete!" (D-08) |
| Other party label (driver view) | "Customer: {commuterName}" |
| Other party label (commuter view) | "Driver: {driverName}" |
| Fare row label | "Estimated Fare" |
| Fare row fallback | "—" (when estimatedPrice is 0 or undefined) |
| Pickup row label | "Pickup" |
| Dropoff row label | "Dropoff" |
| Duration row label | "Duration" |
| Duration fallback | "—" (when timestamps missing) |
| Other party name fallback | "—" (when name unavailable) |
| Empty state heading | Not applicable — screen only appears when trip data is live (trip.status === 'completed') |
| Error state | Not applicable — no async operations on this screen; all data is pre-loaded from existing hooks |
| Destructive confirmation | None — no destructive actions on this screen (D-06: no rating; payments deferred) |

Address display: Full address string from trip.pickupAddress / trip.dropoffAddress. Use numberOfLines={2} with ellipsizeMode="tail" on address value Text components. No truncation logic in code.

Duration display: Format from formatDuration() helper — "X min" (under 60 min) or "Xh Ym" / "Xh" (60+ min). Calculated from completionTime - startedAt (fallback: completionTime - startTime).

Price display: "$X" formatted string. Guard: `trip.estimatedPrice ? \`$\${trip.estimatedPrice}\` : '—'`

**Source:** CONTEXT.md D-03, D-04, D-05, D-06, D-07, D-08; RESEARCH.md Open Questions 2 (address display); RESEARCH.md Pitfall 5 (estimatedPrice guard).

## Layout Contract

### TripCompletionScreen layout (top to bottom, vertically centered):

```
[Modal, animationType="fade", onRequestClose={onDone}]
  [View — flex: 1, backgroundColor: #FFFFFF, alignItems: center, justifyContent: center, paddingHorizontal: 32]
    [Animated.View — spring scale 0→1 on mount]
      [Ionicons checkmark-circle, size: 80, color: #34C759]
    [Text — Display — role-specific header]           marginTop: 24
    [Text — Heading — "{Label}: {otherPartyName}"]    marginTop: 8
    [View — summary card]                              marginTop: 32, width: '100%', backgroundColor: #F5F5F5, borderRadius: 12, padding: 16
      [SummaryRow — "Estimated Fare" / value]
      [Divider — height: 1, backgroundColor: #E0E0E0]
      [SummaryRow — "Pickup" / address, numberOfLines={2}]
      [Divider]
      [SummaryRow — "Dropoff" / address, numberOfLines={2}]
      [Divider]
      [SummaryRow — "Duration" / formatted duration]
    [TouchableOpacity — Done button]                  marginTop: 32, width: '100%'
      backgroundColor: #1565C0, borderRadius: 12, paddingVertical: 16, alignItems: center
      [Text — "Done", fontSize: 16, fontWeight: '600', color: #FFFFFF]
```

SummaryRow layout: flexDirection: 'row', justifyContent: 'space-between', alignItems: 'flex-start', paddingVertical: 12
- Label: fontSize 14, color #666666, flex: 0
- Value: fontSize 15, color #000000, flex: 1, textAlign: 'right', numberOfLines per field

### Animation contract:

- **Trigger:** visible prop becomes true
- **Reset:** scaleAnim.setValue(0) before starting (handles re-open)
- **Animation:** Animated.spring to scaleAnim = 1, tension: 200, friction: 8, useNativeDriver: true
- **Total duration:** ~350ms (spring physics; no explicit duration)
- **Modal entry:** animationType="fade" (React Native built-in, ~300ms)

**Source:** RESEARCH.md Pattern 2 (animation); ActiveTripSheet.tsx ProgressStep (identical spring params); FindingDriverModal.tsx (modal layout structure, button styles).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Screen appears | Modal fades in. Checkmark springs in from scale 0 to 1. |
| "Done" button tap | Calls onDone prop. Parent clears showCompletion, clears activeTripId/requestId, restores driver availability (driver only). |
| Back gesture (iOS swipe-down) | Modal fires onRequestClose → calls onDone. Same cleanup path as Done button. |
| Android back button | Modal fires onRequestClose → calls onDone. Same cleanup path. |
| Cancelled trip | No completion screen shown. Cancelled trips clear activeTripId immediately (unchanged from current behavior). |

**Source:** CONTEXT.md D-05 ("back gesture also works"); RESEARCH.md Pitfall 4 (back gesture bypasses Done handler — solution: onRequestClose={onDone}).
