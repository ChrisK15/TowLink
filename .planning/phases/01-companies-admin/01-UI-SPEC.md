## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native StyleSheet only |
| Preset | not applicable |
| Component library | React Native core (`View`, `Text`, `TouchableOpacity`, `Modal`, `Switch`, `ScrollView`, `FlatList`) |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge internal padding (vertical) |
| sm | 8px | Sibling element gap in a row, chip horizontal padding |
| md | 16px | Card internal padding, screen horizontal margin (default) |
| lg | 20px | Card internal padding where breathing room is needed (matches existing cards) |
| xl | 24px | Section vertical separation |
| 2xl | 32px | Bottom safe-area padding, header-to-content gap |
| 3xl | 48px | Not used in this phase |

Exceptions: Touch targets (icon buttons, swipe actions) must be minimum 44px tall — this is not a spacing token but a minimum height constraint on interactive elements.

## Typography

Four sizes, two weights. Matches existing codebase conventions extracted from `ActiveTripSheet.tsx`, `RequestPopup.tsx`, and `RoleCard.tsx`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px | 400 (regular) | 1.4 | Badge labels, timestamp text, "N drivers online" count label |
| Body | 14px | 400 (regular) | 1.5 | Row secondary text, driver email in roster, job address in list row |
| Label | 16px | 600 (semibold) | 1.4 | Row primary text, driver name, tab bar label, section header, form field label, interactive button labels, status text |
| Heading | 20px | 600 (semibold) | 1.2 | Screen title, tab heading ("Jobs", "Drivers"), form screen heading ("Add Driver") |

Weight palette: regular (400) and semibold (600). These are the only two weights used across all type roles in this phase.

## Component Inventory

New components required by this phase. Reuses React Native primitives and established patterns from existing codebase. No new third-party packages.

### Screens

| Screen | Route | Description |
|--------|-------|-------------|
| Admin Jobs Tab | `app/(admin)/index.tsx` | Real-time list of active jobs; each row has status badge + commuter name + service type + assigned driver |
| Admin Drivers Tab | `app/(admin)/drivers.tsx` | Roster of company drivers with online/offline badge; swipe-left reveals Deactivate button |
| Add Driver Sheet | Bottom sheet or modal | Form: single email input + "Add Driver" CTA; opened from Drivers tab header button |

### Row Patterns

**Job row** (Jobs tab):

The primary focal point on the Jobs tab is the colored status badge chip on the left of each row — it communicates job urgency and state at a glance before the user reads any text.

- Left: colored status badge chip (text: status label) — primary visual anchor
- Middle: commuter name (Label weight) + service type (Body)
- Right: assigned driver name or "Unassigned" in muted text
- Height: 64px minimum (touch target)
- Separator: 1px `#E0E0E0` divider

**Driver row** (Drivers tab):
- Left: 36px avatar circle with initials (Body, semibold)
- Middle: driver name (Label weight) + email (Caption, muted)
- Right: Online/Offline status dot + chip label
- Swipe-left: 44px-tall red "Deactivate Driver" action cell
- Height: 64px minimum

### Status Badges

Pill-shaped badge (border-radius 12, paddingHorizontal 8, paddingVertical 4):

| Status | Background | Text Color | Label |
|--------|-----------|------------|-------|
| pending | `#FFF3E0` | `#E65100` | Pending |
| claimed | `#E3F2FD` | `#1565C0` | Claimed |
| en_route | `#F3E5F5` | `#6A1B9A` | En Route |
| arrived | `#E8F5E9` | `#2E7D32` | Arrived |
| in_progress | `#E8F5E9` | `#2E7D32` | In Progress |
| completed | `#F5F5F5` | `#666666` | Completed |
| cancelled | `#FFEBEE` | `#C62828` | Cancelled |

### Driver Online/Offline Chip

Dot (12x12, borderRadius 6) + text label beside it. No background fill on the chip itself — dot color communicates state.

- Online: dot `#34C759`, text "Online", text color `#34C759`
- Offline: dot `#8E8E93`, text "Offline", text color `#8E8E93`

## Interaction Contracts

### Tab Navigation (Admin Dashboard)

- Two tabs: "Jobs" (default on entry) and "Drivers"
- Tab icons: `briefcase.fill` (Jobs) + `person.2.fill` (Drivers) via `expo-symbols` on iOS, Ionicons equivalents on Android
- Tab switching: no animation delay — standard Expo Router `Tabs` behavior
- Active tab tint: `#007AFF` (matches `Colors.light.tint` in `constants/theme.ts`)

### Real-Time List Updates

- Lists use `FlatList` with a Firestore listener hook (`use-company-jobs.ts`, `use-company-drivers.ts`)
- New items animate in with `Animated.FadeIn` (100ms) — consistent with existing app pattern
- No skeleton loaders: show an `ActivityIndicator` (centered, color `#007AFF`) while the initial snapshot loads
- After first data arrives, `ActivityIndicator` disappears and list renders

### Swipe-to-Deactivate (Drivers tab)

- Swipe direction: left (left-to-right reading order; destructive action revealed on right)
- Swipe threshold: 80px to reveal action cell; beyond 160px = auto-trigger
- Action cell: `#FF3B30` background, white "Deactivate Driver" label (16px, semibold), 44px min height
- On trigger: show `Alert.alert()` confirmation before committing (see Copywriting Contract below)
- Deactivated drivers: remain visible in list with dimmed row (`opacity: 0.5`) and "Deactivated" chip replacing Online/Offline chip. Not hidden (Claude's discretion, rationale: admin needs audit trail of who was deactivated)

### Add Driver Flow

- Trigger: "+" `TouchableOpacity` button in Drivers tab header (right side), 44px touch target, `accessibilityLabel="Add driver"`
- Opens: `@gorhom/bottom-sheet` (already installed) with single-field form
- Form field: email text input (`autoCapitalize: 'none'`, `keyboardType: 'email-address'`, `autoComplete: 'email'`)
- CTA: "Add Driver" button (full-width, `#007AFF`, disabled until valid email format entered)
- On success: dismiss sheet, show brief success banner ("Driver added. They can now sign up with this email."), list refreshes via listener
- On error: inline error text below input field (14px, `#FF3B30`) — no Alert

### Loading States

- List loading: centered `ActivityIndicator` (`color: '#007AFF'`, `size: 'large'`) replacing the list
- Button loading: `ActivityIndicator` (`color: '#FFFFFF'`, `size: 'small'`) inside the button, button disabled, opacity 0.6
- Swipe action loading: none — action is fire-and-forget; list updates in real-time

### Error States

- Firebase operation failure: `Alert.alert('Error', error.message)` — consistent with existing codebase pattern (`try-catch` with `Alert.alert`)
- Empty list (no jobs yet): centered empty state view (see Copywriting Contract)
- Invalid email in Add Driver form: inline field-level error text, not Alert
