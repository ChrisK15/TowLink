<objective>
Wire the admin role into AuthContext and the root layout so that when a user with role='admin' signs in, they land on the admin dashboard instead of falling through to the commuter login.

Purpose: Without this, even after seeding a Firestore user doc with role='admin', the admin would be redirected to commuter-login (the role=null fallthrough). This plan creates the routing plumbing that all admin screens depend on.

- `app/_layout.tsx` — added admin redirect case and `(admin)` Stack.Screen registration
</objective>
