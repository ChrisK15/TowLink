## Spacing Scale

Declared values (multiples of 4). Sourced from audit of ActiveTripSheet, CommuterTripSheet, RequestPopup:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot spacing, thin divider margin |
| sm | 8px | Button row gaps, contact button gaps, badge padding |
| md | 16px | Sheet horizontal padding, card padding, button vertical padding |
| lg | 24px | Header padding, infoRow vertical padding |
| xl | 32px | Sheet bottom margin (safe area clearance) |
| 2xl | 48px | — |

Exceptions:
- Header/infoRow padding in existing components currently renders at 20px — retain as-is to avoid visual regression; do not use 20px for any new elements. New elements use `lg` (24px).
- Drag handle touch target: `paddingVertical: 12` (non-scale, preserves existing hit area — do not change)
- Touch targets for contact/action buttons: minimum 40×40px (existing pattern: contactButton 40×40, initialsCircle 44×44)

## Typography

Sourced from StyleSheet audit across ActiveTripSheet, CommuterTripSheet, RequestPopup. React Native uses numeric sizes (no rem).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 400 (normal) | 1.4 |
| Body | 14px | 400 (normal) | 1.5 (lineHeight: 21 where explicit) |
| Heading | 17px | 700 (bold) | 1.3 |
| Display | 24px | 700 (bold) | 1.2 |

Notes:
- Label (12px/400): section labels, status badge text, vehicle sub-labels, location category labels. Status badges previously rendered at 13px — absorb into 12px Label role; visual difference is imperceptible.
- Body (14px/400): info labels, progress step labels, address text, distance text in InstructionCard, "Open in Maps" button text.
- Heading (17px/700): driver name, commuter name in sheet rows, InstructionCard instruction text. InstructionCard previously specified 16px/600 — use 17px/700 (Heading role) to stay within the 4-size / 2-weight contract.
- Display (24px/700): ETA value, fare amount, request popup title. Sizes previously ranging 20–24px are unified to 24px for Display.
- Two weights only: 400 and 700. Previously used 500 (medium) and 600 (semibold) are replaced — use 400 where subtle weight is desired, 700 where emphasis is needed.

## Color

Sourced from constants/theme.ts and StyleSheet audit. This project does not use a single accent; it uses role-based colors per user type. The contract codifies this explicitly.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Sheet backgrounds, card backgrounds, modal backgrounds |
| Secondary (30%) | #F8F8F8 | Info cards, progress step containers, vehicle cards, page background (#F5F5F5 ≈ same) |
| Accent — Driver | #34C759 | Driver primary action button (advance trip: "I've Arrived", "Start Service", "Complete Trip"), progress dot active state (driver), fare value text, checkmark icon |
| Accent — Commuter | #1565C0 | Commuter status banner text/dot, progress dot active state (commuter), initialsCircle background (commuter), safety banner icon/title, live badge text |
| Accent — System | #007AFF | Accept Request button (RequestPopup), countdown timer, progress bar |
| Destructive | #FF3B30 | "Cancel Job" button (driver, en_route only), "Cancel Trip" button (commuter), expiring-soon timer color |

Accent reserved for:
- **#34C759**: Driver advance-trip action button background ONLY, active progress dots on driver sheet, fare/earnings figures
- **#1565C0**: Commuter status/live indicators ONLY, commuter initials avatar background, safety info elements
- **#007AFF**: Accept Request button and countdown progress bar ONLY
- **#FF3B30**: Destructive cancel actions ONLY — never used for non-destructive UI

New elements in Phase 4 follow the same color roles:
- Instruction card banner (driver navigation): white background (#FFFFFF), text #000/#333, icon Ionicons in #34C759
- Route polyline — driver view: #34C759 (matches driver accent, strokeWidth: 4)
- Route polyline — commuter view: #1565C0 (matches commuter accent, strokeWidth: 4)
- Driver location marker on commuter map: #34C759 circle with white truck icon
- "Open in Maps" button: system blue #007AFF text, no background (tertiary text button)
- Location permission denied state: neutral gray #666 body text, #007AFF link to Settings

## Component Inventory

New components to build in this phase. Follows existing React Native StyleSheet pattern.

### InstructionCard (new)

Overlay banner pinned to top of driver MapView (above the sheet). Displays next-turn maneuver text.

- Position: absolute, top: 60 (below status bar), left: 16, right: 16
- Background: #FFFFFF, borderRadius: 12, padding: 16
- Shadow: elevation 4 (Android), iOS shadowColor '#000' / opacity 0.12 / radius 8
- Layout: row — turn icon (Ionicons, 24px, color #34C759) | instruction text (17px/700, Heading role) | distance text (14px/400, Body role, color #666)
- Step-change animation: fade out old text / fade in new text using `Animated.timing`, 200ms
- When no active trip: hidden (do not render)

### RoutePolyline extension (existing react-native-maps Polyline)

No new component — use `<Polyline>` from react-native-maps.

- Driver screen: coordinates from Directions API response, strokeColor #34C759, strokeWidth 4
- Commuter screen: coordinates from Directions API response, strokeColor #1565C0, strokeWidth 4
- Stage switch (en_route → in_progress): animate polyline swap — 300ms fade via Animated opacity wrapper

### DriverMarker (new, commuter screen)

Custom MapView marker showing driver's live position.

- Outer circle: 32×32, borderRadius 16, backgroundColor #34C759, border 2px white
- Inner icon: Ionicons "car" size 18, color white
- Jump vs smooth: animate between location updates using `Animated.timing` on coordinate (250ms ease-out) — Claude's discretion per CONTEXT.md D-07 decision

### CancelJobButton extension (in ActiveTripSheet)

Shown only when `trip.status === 'en_route'`. Red button at bottom of ActiveTripSheet, above actionButton.

- Style: matches existing cancelButton pattern from CommuterTripSheet exactly
- backgroundColor: #FF3B30, paddingVertical: 16, borderRadius: 12, marginHorizontal: 16, marginBottom: 8
- Text: "Cancel Job", color white, fontSize 14 (Body role), fontWeight '700'
- Confirmation: native Alert (matches existing pattern in CommuterTripSheet.handleCancelTrip)

### OpenInMapsButton (in ActiveTripSheet)

Text button near pickup address in ActiveTripSheet info card.

- Style: no background, paddingVertical: 4
- Text: "Open in Maps", color #007AFF, fontSize 14 (Body role), fontWeight '400'
- Position: below pickupAddress infoRow

## Interaction Contracts

### Driver — Trip Lifecycle

| Current Status | Action Button | Result |
|----------------|--------------|--------|
| en_route | "I've Arrived" | → arrived |
| arrived | "Start Service" | → in_progress |
| in_progress | "Complete Trip" | → completed |
| en_route | "Cancel Job" | Alert confirm → cancelled → re-dispatch |

Cancel is only visible during `en_route`. Once status advances to `arrived`, cancel button disappears permanently.

### GPS-to-Step Matching (Claude's discretion — prescriptive recommendation)

- Advance to next instruction step when driver is within **30 meters** of the next step's start location
- Use `getDistanceInKm()` from services/geoLocationUtils.ts converted to meters
- On wrong turn (driver deviates >150m from current step): re-call Directions API with current position to get new route

### Location Permission Denial (MAP-03)

- On denial: Alert with title "Location Required", body "TowLink needs your location to show directions and update your position for the commuter. Go to Settings → Privacy → Location Services to enable.", two buttons: "Cancel" (dismiss) and "Open Settings" (Linking.openURL('app-settings:'))
- Fallback: if location unavailable, instruction card shows "Enable location to see directions" in #666
- Do not crash — wrap watchPositionAsync in try/catch, silently stop if permission denied mid-trip

### ETA Animation (commuter screen)

- ETA value updates every 30 seconds (D-11 decision)
- Animate number change: fade out old value (150ms), update text, fade in new value (150ms) using `Animated.timing`
- Show "-- min" when ETA not yet loaded (not null/undefined display)

### Bottom Sheet Behavior (existing pattern — no change)

- Driver sheet: collapsed 15% screen height, expanded 75% — unchanged from existing ActiveTripSheet
- Commuter sheet: collapsed 11% screen height, expanded 75% — unchanged from existing CommuterTripSheet
- Instruction card must not overlap with collapsed sheet — card is at top of screen, sheet at bottom; no conflict at default sizes
